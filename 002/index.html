Formatos de Imagens

Os formatos de imagens mais usados para sites sao: PNG e JPEG.

A principal característica do PNG - e que o diferencia do JPEG - é a capacidade de configurar a opacidade de cada pixel (deixá-lo transparente ou com transparência limitada)

Já a vantagem do JPEG é gerar arquivos muito pequenos e que ocupam pouco espaço em disco. Porém, cuidado para nao compactar demais e perder totalmente a qualidade da imagem

De forma resumida, na hora de escolher o formato de imagem para o seu site, opte sempre pelo formato JPEG com uma compactação entre 30% e 50%. O formato PNG só deverá ser usado quando precisarmos de transparência na foto.

Tamanho das imagens

As imagens devem possuir o tamanho certo para cada situacao que queremos executar. Nao podemos simplesmente escolher uma imagem gigante porque a resolucao dela será melhor, mesmo porque o tamanho dela será muito grande, ocupando um espaco maior na memória.
Uma ideia de largura máxima para uma imagem seria 1500px de largura, mas normalmente é usada a largura de 200 a 500px. Já a resolucao pode ficar entre 50 a 80 pixels/in.

A melhor forma de trabalhar com imagens é redimensionar ela antes de colocar no arquivo HTML, com softwares como o PthotoShop ou o Gimp, pois assim, também é alterado o tamanho do arquivo, o deixando mais leve.

Formatacao de Textos

A maior mudança da HTML4 para a HTML5 é o acréscimo da semântica aos elementos.

Na HTML4, tínhamos tags como <b> que colocava um texto em negrito, <u> que colocava o termo sublinhado e <blink> que fazia o texto piscar. Essas eram tags que representavam apenas uma forma.

Já a HTML5 chegou com o conceito de valorizar a semântica, logo suas tags tentam levar um significado embutido muito forte.O intuito é deixar a apresentação gráfica por conta das CSS .

Negrito e Itálico

Para usar o negrito e o itálico recomendamos que você passe a usar as tags <strong> e <em> para realizar essas mesmas formatações visuais, só que agora com sentido.

A tag <strong> significa que o termo delimitado possui força dentro da frase, pois queremos dar destaque. Logo, ele aparecerá em negrito.
Já a tag <em> significa que queremos dar ênfase (do Inglês emphasis) ao termo. Logo, ele aparecerá em itálico. Note que, ao usar <strong> e <em> no lugar de <b> e <i>, damos mais significado aos nossos termos e conteúdos.

Como eles vão ser representados visualmente (forma), vai depender das nossas folhas de estilo CSS.

Trechos de código-fonte

Para representarmos um código na nossa página, usamos a tag <code>. A principal vantagem no uso dessa tag é a o valor semântico que ela representa, indicando ao navegador que se trata de um código de computador.
Porém, existe também um efeito visual, pois as letras ficam no modo mono-espaçadas (monospace), o que facilita bastante a leitura do código.

E, como já sabemos, o ENTER nao tem validade quando escrevemos em HTML, por isso usamos tags como <br>.
No caso de escrever um código, isso nao seria diferente. Por isso, vamos usar uma outra tag HTML chamada <pre>, que mantém o texto pré-formatado, exatamente da mesma maneira na qual ele foi digitado, incluindo quebras de linhas, espaços e tabulações.

Veja o ex. de um código em C#:


    int idade = 24;
    string nome = "Isabele";
    console.WriteLine($"Meu nome é {nome} e tenho {idade} anos.");

“Citações”

Normalmente, uma citação aparece entre aspas ou com as margens deslocadas em relação ao texto.

Citações simples

Para criar uma citação em HTML, podemos usar a tag <q> (do Inglês quote, que significa citar). O texto que estiver entre <q> e </q> já vai receber automaticamente as aspas, mas não terá nenhum deslocamento. Essa técnica é mais usada quando queremos uma citação no meio de um parágrafo.

Exemplo:
Como diria o pai do Ricardo, amigo do prof. Guanabara: O computador é um burro muito rápido.

Citações completas

Também podemos criar citações mais longas (em bloco) e que tenham um parágrafo só para si. Nesse caso, colocaremos tudo dentro de <blockquote> e </blockquote> e o texto ganha um recuo automaticamente.

Podemos também colocar um link para o texto original, usando o parâmetro CITE dentro da tag.

Abreviações

Abreviações sao bem importantes de serem usadas, pois serve para entendermos o que cada letra de uma abreviaçao significa.
Por exemplo: Estou gostando muito de aprender HTML e CSS. É muuuito legal!

Para fazermos isso, usamos a tag <abbr>. Ela já bem com o parametro TITLE para preenchermos com o significado da abreviacao.

Texto invertido

Não entendeu nada desse texto? Pois tente ler da última letra até a primeira.

Pois, é... fazemos isso usando a <bdo>. BDO significa bi-directional override. Ao usar essa tag, já aparece o parâmetro DIR para indicar uma das duas direções possíveis: 


    rtl = da direita para a esquerda (right-to-left) 
    ltr = da esquerda para a direita (left-to-right)
